# فروش ChronoScalp — لایسنس / اشتراک + IB بروکر

این سند مدل کسب‌وکار پشتیبانی‌شده در نرم‌افزار را توضیح می‌دهد.

---

## مدل درآمد (پشتیبانی‌شده)

| منبع درآمد | چگونه | در نرم‌افزار |
|------------|--------|----------------|
| **اشتراک / لایسنس** | مشتری کلید ماهانه/سالانه می‌خرد | ✅ پنل + `manage_licenses.py` |
| **IB بروکر** | مشتری با لینک معرفی شما حساب باز می‌کند؛ بروکر به شما کمیسیون می‌دهد | ✅ بخش IB در پنل + `saas.ib_referral_url` |
| ۱٪ اتومات از سود معامله مشتری | از حساب بروکر مشتری به حساب شما | ❌ پشتیبانی نمی‌شود |

---

## برای فروشنده (شما)

### ۱. رمز ادمین
در `.env`:
```env
LICENSE_ADMIN_SECRET=یک-رشته-بلند-تصادفی
```

### ۲. صدور لایسنس
**از پنل:** `streamlit run scripts/app.py` → «مدیر لایسنس»  
**یا CLI:**
```bash
python scripts/manage_licenses.py issue --tier monthly --email customer@mail.com --name "Ali"
```

پلن‌ها: `trial` (۷ روز) · `monthly` · `yearly` · `lifetime`

### ۳. لینک IB
لینک Partner/IB خود را در `config/settings.yaml` بگذارید:
```yaml
saas:
  ib_referral_url: "https://www.oanda.com/...?ref=YOURCODE"
```

### ۴. اجبار لایسنس
```yaml
licensing:
  require_license: true   # ربات بدون کلید استارت نمی‌شود
```
برای استفاده شخصی بدون فروش: `require_license: false`

---

## برای مشتری (تجربه کاربری)

1. `scripts\start.bat` یا `streamlit run scripts/app.py`
2. **لایسنس** — کلید را وارد و فعال کند
3. **اتصال بروکر** — OANDA یا MT5 → تست اتصال → ذخیره
4. **کنترل ربات** — Paper استارت (بعداً Live)
5. **مانیتورینگ** — وضعیت و لاگ

همه چیز از UI؛ نیازی به ویرایش دستی YAML برای مشتری معمولی نیست.

---

## جریان پیشنهادی فروش

```
مشتری پول اشتراک می‌دهد
        ↓
شما کلید لایسنس صادر می‌کنید
        ↓
مشتری کلید را در پنل فعال می‌کند
        ↓
با لینک IB شما در OANDA ثبت‌نام می‌کند (اختیاری ولی سودآور)
        ↓
بروکر را از پنل وصل می‌کند → Paper → Live
```

## امنیت

- `LICENSE_ADMIN_SECRET` را هرگز به مشتری ندهید
- کلیدهای API مشتری فقط در `.env` محلی‌شان است
- فایل‌های `data/licenses/` و `data/user/` در git نیستند
- لغو لایسنس: `python scripts/manage_licenses.py revoke KEY`
